  <action>
Create the Astro project and configure Tailwind v4 with the CreativAudio design system.

1. **Create Astro project** using `npm create astro@latest` with the minimal template. Run from the project root (C:/Users/zepid/Documents/CreativAudio). Since the directory already has .planning/, use the `--template minimal` flag and install into the current directory. If the CLI prompts for options, accept defaults (TypeScript: yes, strict). Verify the installed Astro version is 5.x (not 6.x alpha).

2. **Install Tailwind CSS v4** and fonts:
   ```
   npm install tailwindcss @tailwindcss/vite
   npm install @fontsource/orbitron @fontsource/inter @fontsource/dm-mono
   ```

3. **Configure astro.config.mjs** — Register @tailwindcss/vite as a Vite plugin. Do NOT use @astrojs/tailwind (deprecated for v4). Do NOT add @astrojs/vercel adapter (not needed for static sites):
   ```js
   import { defineConfig } from "astro/config";
   import tailwindcss from "@tailwindcss/vite";
   export default defineConfig({
     vite: {
       plugins: [tailwindcss()],
     },
   });
   ```

4. **Create src/styles/global.css** with Tailwind import and @theme design tokens:
   - `@import "tailwindcss";`
   - Define in @theme block: --color-accent (#FF5C00), --color-accent-light (#FF7A33), --color-accent-dark (#CC4A00), --color-bg (#FFFFFF), --color-surface (#F9FAFB), --color-border (#E5E7EB), --color-text (#1A1A1A), --color-text-muted (#6B7280)
   - Define fonts: --font-heading (Orbitron), --font-body (Inter), --font-mono (DM Mono)
   - Add base body style setting font-family to var(--font-body)
   - Do NOT use @tailwind directives (v3 pattern). Do NOT create tailwind.config.js.

5. **Create src/layouts/Layout.astro** — Import Fontsource CSS files in frontmatter:
   - Orbitron: 400, 700, 900
   - Inter: 400, 500, 600, 700
   - DM Mono: 400
   - Import global.css
   - HTML shell with lang="en", charset, viewport meta, description meta, title prop, favicon link
   - Body with classes: `bg-bg text-text font-body`

6. **Create src/pages/index.astro** — A verification page that exercises all design tokens:
   - Import and wrap in Layout component with title "CreativAudio - Orbital Synthesizer"
   - Show "CreativAudio" heading in font-heading (Orbitron), bold
   - Show "Orbital Synthesizer" in font-mono (DM Mono), accent color
   - Show body text in default font (Inter)
   - Show a filled rectangle/button element with bg-accent to verify #FF5C00
   - Show text-muted, bg-surface, border examples so all tokens are visually verifiable

7. **Create public/favicon.svg** — A simple orange circle SVG (16x16 viewBox, filled #FF5C00) as a placeholder favicon.

Set up version control and deploy the site to Vercel.

1. **Verify .gitignore** exists and includes: node_modules, dist, .astro, .vercel, .env. Astro's scaffolder should have created one, but verify it covers these. Add .planning/ exclusion is NOT needed (planning docs should be tracked).

2. **Create initial commit:**
   ```
   git add .
   git commit -m "feat: scaffold Astro 5.x + Tailwind v4 with CreativAudio design tokens"
   ```

3. **Create GitHub repo and push** using gh CLI:
   ```
   gh repo create CreativAudio --public --source=. --push
   ```
   If the repo name is taken or the user prefers private, adjust accordingly. The key is getting code on GitHub.

4. **Deploy to Vercel** using the Vercel CLI. First check if vercel CLI is available (`npx vercel --version`). Then:
   ```
   npx vercel link --yes
   npx vercel deploy --prod
   ```
   If vercel CLI is not authenticated, this will become a checkpoint. For a static Astro site, Vercel auto-detects the framework — no special configuration needed. Do NOT install @astrojs/vercel adapter.

5. **Record the deployment URL** from the Vercel output. This URL will be used in the verification checkpoint.
